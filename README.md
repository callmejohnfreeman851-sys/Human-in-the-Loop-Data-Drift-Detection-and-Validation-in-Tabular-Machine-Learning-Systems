## 📌 Project Overview

In real-world fraud detection systems, models trained on historical data degrade silently as transaction patterns evolve — a phenomenon called **data drift**. This project addresses that problem with a **Human-in-the-Loop (HITL)** pipeline:

1. 📊 Detect drift statistically using multiple complementary tests
2. 🧑‍💼 Present drifted features to a **human expert** for business validation
3. ✅ Retrain **only** when expert confirms the drift is business-critical
4. 📈 Measure and visualise performance recovery after retraining

## 📊 Key Results

### 🔍 Drift Detection
- **KS Test** (Bonferroni-corrected): Identified statistically significant drift in numerical features
- **Chi-Square Test** (Bonferroni-corrected): Detected category distribution shifts
- **PSI** (Fixed-width bins): Classified features as Stable / Monitor / Unstable

### 📉 Performance Degradation (Drift Impact)
| Stage | ROC-AUC |
|-------|---------|
| Baseline on Reference Data | ~0.97+ |
| Baseline on Candidate Data | Degraded |
| **Retrained on Candidate Data** | **Recovered ✅** |

> *Exact values will appear after running the notebook with your data.*

### 🧑‍💼 Human-in-the-Loop Widget
The expert review interface (Phase 6) presents each drifted feature with:
- KS Statistic, PSI value, Chi-Square result
- Drift direction (↑ increase / ↓ decrease)
- Multi-test confirmation flag ⚠
- YES / NO toggle for business-critical decision

This project uses the **IEEE-CIS Fraud Detection** dataset from Kaggle.

## 🧪 Statistical Methods Used

### Drift Detection
- **Kolmogorov-Smirnov Test** — Non-parametric test for numerical distribution shift
- **Chi-Square Test** — Frequency distribution change in categorical features  
- **Population Stability Index (PSI)** — Magnitude of distribution shift (fixed-width bins)
- **Bonferroni Correction** — Multiple testing correction to control false positives

### Model Evaluation
- **Bootstrap Confidence Intervals** — Statistical significance of ROC-AUC degradation
- **F1-Score Maximisation** — Optimal threshold selection for imbalanced data
- **ROC-AUC, Precision, Recall, F1** — Full performance comparison across pipeline stages

## 💡 Why Human-in-the-Loop?

Not all statistical drift is operationally meaningful:
- A feature may drift statistically but have **no fraud detection impact**
- Drift caused by **seasonal patterns** ≠ drift requiring retraining
- Domain experts can distinguish **business-critical** shifts from noise

This HITL approach is more reliable than fully automated retraining pipelines.
